非常好的问题！Python 函数参数的顺序是有严格规定的，**必须按照特定顺序书写**，否则会报 `SyntaxError`。

---

### ✅ 正确的参数定义顺序（从左到右）：

```python
def func(
    pos_only,                     # 1. 位置参数（仅位置）
    /,                            # （可选）分隔符：左边只能用位置传参（Python 3.8+）
    pos_or_kwd,                   # 2. 位置或关键字参数（最常见）
    *,
    kwd_only,                     # 3. 仅关键字参数（必须用关键字传）
    *args,                        # 4. 可变位置参数（收集多余位置参数）
    **kwargs                      # 5. 可变关键字参数（收集多余关键字参数）
):
    pass
```

但在**日常使用中**（不涉及 `/` 和 `*` 的高级用法），我们通常只关心以下 **5 类参数的顺序**：

---

### 📌 日常开发中最常见的参数顺序（必记！）：

| 顺序 | 参数类型             | 是否有默认值 | 示例 |
|------|----------------------|--------------|------|
| 1️⃣   | **普通位置参数**     | ❌ 无默认值   | `a, b` |
| 2️⃣   | **带默认值的参数**   | ✅ 有默认值   | `c=10, d="hello"` |
| 3️⃣   | **可变位置参数**     | —            | `*args` |
| 4️⃣   | **仅关键字参数**（可选）| 可有可无     | `*, e, f=5`（注意 `*` 后面的参数只能用关键字传）|
| 5️⃣   | **可变关键字参数**   | —            | `**kwargs` |

> 💡 **核心规则**：**所有无默认值的参数必须出现在有默认值的参数之前！**

---

### ✅ 正确示例（推荐写法）：
```python
def example(a, b, c=10, d=20, *args, e, f=30, **kwargs):
    print(a, b, c, d, args, e, f, kwargs)
```

调用方式：
```python
example(1, 2, 3, 4, 5, 6, e=7, f=8, x=9, y=10)
# a=1, b=2, c=3, d=4, args=(5,6), e=7, f=8, kwargs={'x':9, 'y':10}
```

> 注意：`e` 是**仅关键字参数**，必须写成 `e=...`，不能靠位置传！

---

### ❌ 错误示例（你遇到的情况）：
```python
def bad_func(a=1, b):  # SyntaxError!
    pass
```
> 报错：`non-default argument follows default argument`  
> 原因：`b` 没有默认值，却跟在有默认值的 `a` 后面。

---

### 🔁 快速记忆口诀：

> **“无默在前，有默在后；星参最后，双星收尾。”**

- **无默**：无默认值参数（必须先写）
- **有默**：有默认值参数（写在无默之后）
- **星参**：`*args`
- **双星**：`**kwargs`

---

### 🧠 补充说明（FastAPI 场景）：
在 FastAPI 中，路径操作函数的参数通常是：
- **Query/Path/Body 等依赖项** → 本质是**带默认值的关键字参数**
- 所以如果你写了 `offset: int = 0`（有默认值），后面就不能再写 `limit: int`（无默认值）

✅ 正确（FastAPI）：
```python
def get_items(offset: int = 0, limit: int = 10):
    ...
```

❌ 错误（FastAPI）：
```python
def get_items(offset: int = 0, limit: int):  # SyntaxError!
    ...
```
